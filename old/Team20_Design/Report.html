Task 3 - Paragraph

We have designed our project to have as little coupling as reasonably possible, and to maximally utilize data coupling. Our data is stored in entity classes and the modules communicate by passing along this data. For example, the provider terminal class reads the provider directory class and the service records class. However, the provider terminal class does not change the data within the directory and records class, thus avoiding content coupling. We have designed our project to maximize the cohesion. Our modules utilize functional cohesion by including methods that only perform one action. For example, our Member Updater class only includes methods that modify the member records in the least invasive way. Also, our classes and methods have been designed with the intent to have independent code to implement each action. Through these actions, we are able to move forward with writing good code, with low coupling and high cohesion.
